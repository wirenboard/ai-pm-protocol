Close the gap: a **state/event-model** that is implemented and justified but **embedded in prose** — status values, transitions, recompute triggers, debounce written as paragraph-form "why" — forces a reader to reverse-engineer "the state machine" from narrative. This slice adds (1) a **conditional `## State model` section** to `docs/architecture.md` — a states×transitions table (+ triggers) that **references** the existing prose and the status taxonomy in `## Behavioral contract`, never re-derives them; and (2) a **conditional `/pm-plan` trigger** that fires when the feature's core **is** a state machine / protocol. It is the direct sibling of the nfr-prompt slice (conditional prompt + conditional architecture section) and reuses that pattern verbatim.

## Scenarios

1. **A state-machine / protocol-bearing feature → the State-model check fires (the missing prompt).** When the orchestrator judges, while planning, that the feature's **core is a state machine or protocol** — a status ladder with defined transitions, a lifecycle with named states + the events that move between them, a wire protocol with message/connection states — `/pm-plan` runs a **conditional State-model check**: it prompts to capture the model (states, transitions, recompute/transition triggers, any debounce) into the new `## State model` section. The check sits beside the existing conditional pre-draft checks (Stack-component, Interaction-scenario, Security-surface, NFR/operational-limits) — same judgment-triggered, proportional shape.

2. **Non-state-bearing feature → silent (proportional).** A feature whose core is **not** a state machine / protocol gets **no** State-model prompt — silent, exactly like the NFR check on a non-scale feature and the Security-surface check on a non-security project. The check is **conditional / proportional, never blanket-mandatory** (the backlog's form-remark; the expert's "conditional" flag). No new mandatory plan section, no hook (the trigger is a semantic judgement a regex cannot make — the same no-hook reality as the NFR/operational-limits and product-readiness checks).

3. **The model lands in a conditional `docs/architecture.md` `## State model` section — by reference, not re-derivation.** The captured model goes to a **new conditional `## State model` section** in `docs/architecture.md` (owned by `pm-architect`, authored/refreshed on the `/pm-plan` "Docs to update" post-coding handoff — the same trigger as a decision record). It is a **states×transitions table + a triggers list** that **references** the existing prose (the "why" justification stays where it is) and **references the status taxonomy in `## Behavioral contract`** (the state *names* live there once — the State model shows the *transitions* between them, it never restates the enum — the same move-not-copy discipline the journeys already follow). The section is **`N/A — <reason>`** for non-state-bearing projects (proportional, like every other conditional section). A transition/trigger the orchestrator cannot determine without real system knowledge is recorded as `[?]` for the PM to confirm — **never invented**.

4. **Already-captured guard (no re-prompt churn).** Like the NFR system-budget guard: once the `## State model` section captures the machine (a non-`N/A`/`[?]` table), the check stays **silent** on the already-modeled states and fires only for a feature that introduces a **new state / transition / trigger** not yet in the section — so a state-bearing project is not re-prompted for its whole model on every later plan.

5. **Structural / proportional, additive, no migration, no hard gate.** The check matches a **judgement** (is the feature's core a state machine?), never a regex; it records into an **existing-owner home** (a new conditional architecture section), not a new parallel mandatory document — the backlog's form-remark verbatim. The architecture template gains one **conditional** section (born `N/A`); existing projects are unaffected until their next `/pm-plan` (a state-bearing feature earns the prompt) and their next architecture refresh (the `N/A` section). **No hard gate** — consistent with the nfr-prompt slice: the prompt degrades silently if the trigger is misjudged, and the home makes a recorded model recoverable at the next refresh (a future "did-it-right" check is the same deferred slice-N candidate the nfr slice named). No `MIGRATIONS.md` / template structural migration.
