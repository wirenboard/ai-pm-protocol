## i18n

The **i18n** module is on, so where the project serves (or plans) more than one locale
the review gains the locale dimension the floor does not name — inert on a
single-locale project: judge the change in the user's locale, not the author's.
`[persona]`: this sharpens judgement, denies nothing.

- `[light]` **Externalization honoured** — a hardcoded user-facing string where the project externalizes, or a translation assembled by concatenation, is a finding.
- `[light]` **Locale-blind formatting is a finding** — a hand-built date, number, or currency format on a user-facing path bypasses the locale APIs the project uses.
- `[rich]` **Plural / expansion / RTL concretely checked** — a hardcoded singular/plural pair, a layout that breaks on text expansion, or an unhandled direction on a named RTL target is a cited finding.
