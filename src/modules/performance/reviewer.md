## Performance

The **performance** module is on, so the review gains the scale dimension the floor's
**Correctness** item does not name: where the change touches a loop, query, or payload
over user-scale data, judge it at the scale the plan named — not at the demo size the
diff was tried at. `[persona]`: this sharpens judgement, denies nothing.

- `[light]` **Unbounded path is a finding** — a query or load over user-scale data with no limit / pagination at the boundary fails at scale by design.
- `[light]` **Perf claim needs numbers** — a performance claim without a measurement is an over-claim and blocks like any honesty over-claim.
- `[rich]` **Scale-shape check** — an N+1 query or an O(n²) pass on a path the plan named as user-scale is a cited finding.
