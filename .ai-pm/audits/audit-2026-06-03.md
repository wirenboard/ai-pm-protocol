# Protocol audit — 2026-06-03

## Summary

Full-scope sweep of the ai-pm-protocol template repo across its entire history (21 feature plans, all squash-merged to `main`). Protocol health is strong: every shipped feature has a plan, all but a small set of pre-protocol bundle plans have matching reviews, the migration catalogue is correctly single-sourced, English-canonical landed cleanly (no Russian output strings remain — only detection signals), and the 71/71 hook test suite passes. The one real gap is **internal docs drift in the protocol's own canon**: `doc/architecture.md` still uses the pre-`pm-*`-rename agent/command names throughout (two of them are dead file references), and `WORKFLOW.md` still calls the *generated* render `docs/product.md` instead of the v2.3 `docs/product-map.md`. Both are exactly the docs-currency drift this protocol exists to catch — applied to itself. No blocking findings.

This report supersedes today's earlier diff-scope audit (0 blocking / 0 notes over the 5-release window); the full sweep crosses older files the narrow window did not, surfacing two stale-name notes plus a stale review-date.

## Scope exceptions (recorded in `doc/architecture.md`, not flagged)

This template repo's "product" is the protocol itself, so by deliberate design it has **no** `.ai-pm/contracts/`, **no** `.ai-pm/state/`, and produces **no** `docs/product-map.md` / `docs/product.md` of its own. It uses `doc/` (singular), `doc/features/` for plans, `.ai-pm/reviews/` for reviews, `.ai-pm/audits/` for audits. The only executable test is `tests/hooks.sh` — **run during this audit: 71/71 pass.** None of the contract-presence / product-map / product.md / state checks apply here; they are skipped per the architecture.md exception, not flagged.

Concretely:

   - **Dead reference** `doc/architecture.md:25` → `.claude/agents/pr-prep.md` (no such file; actual is `pm-pr-prep.md`).
   - **File-layout table** `:140` lists "Seven persona files (`architect`, `auditor`, `coder`, `docs-extractor`, `pr-prep`, `reviewer`, `stack-researcher`)" — wrong names: `docs-extractor`→`pm-legacy-reader`, `reviewer`→`pm-plan-checker`, none carry the `pm-` prefix.
   - **File-layout table** `:141` lists "Five slash-command procedures (`audit`, `bootstrap`, `fixup`, `plan-feature`, `research`)" — `plan-feature`→`pm-plan`, none carry the `pm-` prefix.
   - Body prose at `:11`, `:42`, `:56–58`, `:62`, `:82`, `:234` uses the old generic names / `/bootstrap` `/plan-feature` slash forms.
   **Why it matters:** `architecture.md` is the protocol's own single source of truth for *why the template is built this way*, and it self-contradicts the shipped `.claude/` files and the already-migrated `WORKFLOW.md`. The two dead `.claude/agents/*.md` references point a reader at files that do not exist. 2. **`WORKFLOW.md:207` still calls the generated render `docs/product.md` — stale vs the v2.3 split.** The paragraph reads "The PM-facing render of all contracts … is `docs/product.md`. It is generated … never hand-filled … `pm-auditor` re-derives `product.md` from source on every audit." Since v2.3 (`product-md-front-door`) the **generated** map is `docs/product-map.md` and `docs/product.md` is the **authored** front door owned by `pm-architect` and never regenerated. The v2.3 feature updated the templates, agents, and `architecture.md` but missed this WORKFLOW.md paragraph. **Why it matters:** `WORKFLOW.md` is the canonical orchestration spec read by every agent and imported into every downstream `CLAUDE.md` (`@.ai-pm/tooling/WORKFLOW.md`) — a downstream agent reading this line would regenerate the *authored* front door, clobbering PM-authored prose; it directly contradicts `pm-bootstrap.md`, `pm-auditor.md`, and the v2.3/v2.6 decisions. **Remediation:** `/pm-plan` or a scoped `/pm-fixup` updating the WORKFLOW.md paragraph to name `docs/product-map.md` as the generated map and `docs/product.md` as the authored front door.

3. **`doc/architecture.md` header `Last full review: 2026-05-30` is stale.** The document carries three architectural-decision subsections dated 2026-06-02/03 (value-first map, Behavioral contract, English-canonical) plus a 2026-06-03 currency note, and git's last edit to the file is 2026-06-03. The header date predates all of them. **Why it matters:** the review-date field is the doc's own currency signal; a stale date understates how recently the canon moved. **Remediation:** `pm-architect` bumps the date when it makes the Note-1 name fixes (same file, same owner — batch them).

4. **Three v2.2.0-bundle plans plus the v2 rewrite carry no `_review.md` (pre-protocol-migration artifact).** `contract-centric-product-map`, `diagnostic-probe-mode`, and `protocol-builtins-realignment` were all shipped in a single squashed commit (`be2dba1 feat: v2.2.0 …`); `template-v2` shipped in `eba8ab5 feat: template v2 — full rewrite`. Not blocking.
