# Protocol gap: authored docs optimize the author, not the reader — bloat neither human nor model holds

## Symptom

A downstream project built under this protocol (`wb-mqtt-matter`) accumulated technical documentation that is **simultaneously too large and too thin**, to the point where neither a human reviewer nor the model can hold it:

- `docs/architecture.md` — 124 KB / 668 lines. Compression ratio 42% of source (above the 10–40% healthy band). One decision section (`### Last-fabric re-pair`, 10.3 KB) carries its load-bearing fact in the first two sentences, then ~8 KB of re-framing, process-history, and a 1.5 KB `Source:` provenance block citing commit hashes and `.ai-pm/` paths that are **outside the reader's readable surface**.
- `docs/stack-notes.md` — 234 KB (0.8× the size of the entire source tree). Its `### matter.js` block is 195 KB **with no `####` substructure**, ordered chronologically by feature date, ~25% noise (investigation-narrative, corrected-record boilerplate kept after resolution, an open work-item). Node density 0.09 idioms / 100 tokens — 3–5× below a usable reference.

Expected: a doc a reviewer can read and a diff a PM can grasp. Actual (the project owner's words): *"вода-вода-вода … человек это не может понять, диффы не может осознать, ну и всё оно уплывает."* And the model's failure mode, named by the owner: *"видит мегабайтный md файл, и даже не читает его нормально, просто дописывает новый кусок"* — append-only growth, never re-read, never restructured.

The distinction the doc-quality report drew (this is *repeats + history + provenance*, not literal *filler*) is true for diagnosis but **does not help the reader** — from the reader's seat the effect is identical: signal is undiscoverable, the artifact drifts.

**Second data point — `nula` (13 source files, 36 KB, young, clean) proves the bloat is structural, not accumulated cruft.** Where wb-mqtt-matter could be dismissed as a large mature project that grew history-cruft, nula cannot: it is tiny, the structure is clean (AD1–AD8, behavioral contract, state model, all sections present), the content is dense and substantive (not filler), the contracts are correctly small (0.7–2.2 KB — two-layer works). And it *still* carries `docs/architecture.md` at 48.5 KB (**135% of the entire source tree**) and `docs/` totalling 170 KB (**4.7× the code**) — **from day one**. The cause is not accumulation: it is that the decision-record format is **verbose by design** (full-prose `Chosen / Why / Rejected / Source / [?]` per decision) with **no density discipline** — nothing tells the author "one line, not one paragraph." Example (nula AD6): *"This is the cost the PM accepted for the shared-codebase economy, and it is the standing obligation recorded as SC12 that reviewers and coders must uphold on every route and data API in the owner and viewer zones."* — one sentence where *"Cost: every owner-zone route/API must enforce the server-side role check (SC12)"* carries the same content. The same provenance-outside-the-reader's-surface pattern (`Source: bootstrap conversation`, 6× `.ai-pm/` refs) and history-in-current-state (`[x] Resolved (2026-06-07)` inline in AD5) appear, milder. So the two projects are **two faces of one gap**: wb-mqtt-matter = *drowned* (accumulated history/provenance/narrative + append-only growth + lost-in-middle); nula = *born bloated* (verbose-by-design format, no density discipline, docs disproportionate to code on a clean young project). Both trace to the single root below.

## Root cause

The protocol is **uniformly optimized for the author and for maintenance**, with **no counter-force for the reader**. Every one of these rules is individually correct, and together they *produce* the bloat:

1. **Provenance / decision-record discipline** → inline `Source:` blocks with commit hashes and `.ai-pm/arch/` references. The references point *outside the downstream reader's readable surface* (the boundary invariant excludes `.ai-pm/tooling`; arch-notes live in `.ai-pm/arch/`), so they are zero-value to the consumer of `docs/` — pure noise.
2. **History preservation in decision-records** (`Supersedes`, `corrects the earlier wording`, redesign narrative) → `docs/architecture.md` carries the *story of how the decision changed* mixed into the *description of current logic*. The current-state reader wants the latter only.
3. **`pm-stack-researcher` mandate to cite canonical sources + confidence tags + spike discipline** → the *investigation trail* (172 `node_modules` traces, `SOURCE-TRACED…`) is dumped into `docs/stack-notes.md` at 3–7× the length of the rule it justifies.
4. **Append-on-touch growth with no compaction trigger.** `/pm-plan`'s stack-check only *appends* a component when touched. **No rule anywhere triggers periodic restructure/compaction** of any authored doc. `pm-auditor` checks staleness and compliance but has **no size / density / lost-in-the-middle dimension** — so unbounded growth is invisible to every gate. This is the mechanism behind the owner's "просто дописывает новый кусок."
5. **Single-source-per-rule + "reference by name, never duplicate"** → optimal for no-drift, but spreads one module's knowledge across 4–8 files with no consolidated per-module home, and forces the reader to chase references.
6. **No format discipline for the load-bearing fact.** Sections lead with framing/context; the fact that breaks if violated sits mid-section (`### Last-fabric re-pair` buries the critical "re-wire ALL observers after node re-init" invariant in the middle of the longest paragraph).

The deeper framing (from the project owner): the protocol does **not neglect** architecture — it has heavy architecture machinery (`pm-architect`, arch-notes, decision-records, state-model / NFR checks, contracts). It **drowns** architecture: the machinery produces *volume*, not *legibility*. Drowned architecture is, from every consumer's seat, the same as neglected architecture — and it forces the exact offloading-to-experts the owner wants to avoid. A doc small enough for a human to hold is also small enough for the model to reason over and for a non-coding PM to engage with — so **compression is the enabling condition for shared architectural understanding, not cosmetics.**

A worked proof: the 10.3 KB `### Last-fabric re-pair` section compresses ~4× (to ~2.6 KB) with **zero lost guarantees** — every `must`/`never`, boundary, and control survives — because ~75% of the volume was meta-wrapping (formulation-history + provenance + process-narrative), not information. (Full before/after recorded in the design discussion that spawned this report.)

## Minimal fixes

Five components; (B) and (D) are the structural core, the rest are format discipline. **Recommended: ship all five as one feature** — they are one tension and fixing only the format rules without the compaction trigger (B) lets drift re-accumulate.

- **(A) Fact-first + current-state-only format rule** for authored docs — *new* `workflow/doc-style.md` topic file (single home), echoed in the `pm-architect` and `pm-stack-researcher` personas. Each section opens with the fact that breaks if the rule is violated; `docs/` carries current state, not the story of how it changed.
- **(B) ⟵ recommended core. A size/density signal + compaction as a first-class operation.** Give `pm-auditor` a doc-frugality dimension (size / node-density / lost-in-the-middle / append-only-flatness), and make *restructure/compaction* a named remediation `pm-architect` runs when it fires — the same move already applied to `WORKFLOW.md` (thin router + on-demand topic files), now extended to project docs. Files: `workflow/review-typology.md` or a new dimension in the auditor + `pm-auditor` / `pm-architect` personas + the `pm-audit` skill remediation list.
- **(C) Provenance = one-line pointer, never an inline block.** `Source:` blocks with commit hashes / `.ai-pm/` paths collapse to a single pointer line (or move to git history / the arch-note, which is provenance's real home). Rule in `workflow/doc-style.md`; `pm-architect` persona echo.
- **(D) ⟵ recommended core. Contracts super-small + mandatory two-layer.** Harden the existing two-layer contract migration from optional to required, with a size ceiling on the PM-facing layer (product language only; machine grammar lives in the referenced architecture layer). Files: `MIGRATIONS.md` (promote the migration), the contract template under `doc/_templates/`, `pm-architect` persona.
- **(E) History/narrative has a home that is not `docs/`.** Investigation-narrative → distilled rule + one `Source:` line (the trace lives in the arch-note / git, not stack-notes). Corrected-records collapse to the final rule once resolved. Rule in `workflow/doc-style.md`; `pm-stack-researcher` persona echo.

**Out of scope of this gap (related, separate):** the PM-comms *stance* shift the owner also raised — the AI actively teaching architecture, offering variants, growing its own model of the software, raising the PM *up* to architecture rather than shielding the PM from it. That is a pm-comms / role-philosophy change, fuzzier and worth its own gap. This gap is the concrete, shippable lever (frugality + structure) that is *also* the enabling precondition for that vision.
